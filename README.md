# 螺旋案例協商推理中醫輔助診斷系統
**Spiral Case-Based Reasoning Clinical Decision Support System**


##  系統概述

本系統為一個的**中醫輔助診斷系統**，核心演算法融合三項核心技術：

### 1. **案例推理 (Case-Based Reasoning, CBR)**
- 從過往的中醫醫療案例庫中檢索相似案例
- 基於歷史案例推導當前診斷
- 提高診斷的循證醫學基礎

### 2. **螺旋式協商推理 (Spiral Case-Based Reasoning, SCBR)**
- 模擬中醫師真實的互動式問診流程
- 多輪交互完善診斷假設
- 逐步收斂到最優診斷結論
- 提升診斷準確性和可靠性

### 3. **大型語言模型 (LLM) 驅動**
- 使用 NVIDIA llama-3.3-nemotron-super-49b-v1.5進行自然語言推理
- 生成結構化的診斷報告與治療建議
- 支持複雜的醫學推導過程

---

##  主要功能

| 功能模組 | 描述 |
|---------|------|
| **患者問診** | Web 介面收集患者主訴與症狀 |
| **案例檢索** | 向量檢索 + 混合搜索找出相似醫療案例 |
| **症狀分析** | AI 解析症狀特徵並啟動推理流程 |
| **交互式追問** | 系統主動提問以收集更多診斷證據 |
| **辨證論治** | 生成中醫辨證分型與治療方案 |
| **安全防護** | Guardrails 過濾不當回應 |
| **專家審核** | 支持醫療專家人工複審診斷結果 |
| **案例保留** | 優質診斷案例自動保存到知識庫 |

##  使用流程

### 三階段互動診斷

#### **第一階段：初始問診**
患者輸入主要症狀，系統進行初步分析

#### **第二階段：交互式追問**
系統根據需要進行 1~3 輪追問以收集更多信息

#### **第三階段：診斷報告生成**
系統生成完整的診斷報告，包括辨證結果、病機分析、治則建議和信心評分

##  關鍵特性

-  **循證診斷** - 基於實際醫療案例
-  **AI 驅動** - llama-3.3-nemotron-super-49b-v1.5 推理
-  **交互式推理** - 真實臨床決策過程
-  **可解釋性** - 詳細推導過程說明
-  **多層安全防護** - Guardrails + 審核
-  **易於部署** - Docker Compose 一鍵啟動
-  **可擴展架構** - 模組化設計
